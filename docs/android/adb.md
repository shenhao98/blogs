当ADB无线连接出现「由于目标计算机积极拒绝，无法连接」错误时，核心原因是**安卓设备未正确监听指定端口**，或电脑与设备的网络/权限不匹配。下面我会拆解错误原因，并给出一步步可操作的解决方案：

### 一、先理解错误本质

「积极拒绝」= 电脑向安卓设备的`IP:端口`发送连接请求，但设备**没有程序在该端口监听**（ADB的tcpip模式未开启/端口被占用/网络不通），直接拒绝了连接。

### 二、分步排查 & 解决（按优先级排序）

#### 步骤1：重新确认核心前提（必做）

1. **电脑和设备必须在同一网络**：
   - 检查：电脑连Wi-Fi（不要连有线）、设备连同一个Wi-Fi；
   - 验证：电脑ping设备IP（Windows：`ping 设备IP`；macOS/Linux：`ping 设备IP`），能ping通才说明网络通。
2. **设备已开启USB调试**：
   - 重新进入「开发者选项」→ 关闭再开启「USB调试」，确保未被系统自动关闭。

#### 步骤2：重新初始化ADB的TCPIP模式（最常见解决方法）

错误的核心原因是「adb tcpip 5555」命令未真正生效，需重新插线执行：

1. 用数据线将设备连到电脑；
2. 打开命令行/终端，执行以下命令（逐行来）：

   ```bash
   # 1. 重启ADB服务，清理异常
   adb kill-server
   adb start-server

   # 2. 查看设备是否被识别（确保有线连接正常）
   adb devices
   ```

   - 若列表中显示设备（状态为device），说明有线正常；
   - 若显示unauthorized：设备端会弹「允许USB调试？」，勾选「始终允许」并确认。

3. **重新设置TCPIP端口**：

   ```bash
   # 先切换到USB模式（重置），再切回TCPIP
   adb usb
   adb tcpip 5555
   ```

   - 成功提示：`restarting in TCP mode port: 5555`；
   - 若提示「error: device not found」：换数据线/换电脑USB口（优先用USB2.0）。

4. 拔掉数据线，重新执行连接命令：
   ```bash
   adb connect 设备IP:5555
   ```

#### 步骤3：排查端口问题（端口被占用/设备未监听）

1. **换一个端口试试**（避免5555被占用）：
   ```bash
   # 插线状态下，改用5556端口
   adb tcpip 5556
   # 拔线后连接新端口
   adb connect 设备IP:5556
   ```
2. **验证设备是否在监听端口**（进阶）：
   - 插线状态下执行：`adb shell netstat -an | findstr 5555`（Windows）/ `adb shell netstat -an | grep 5555`（macOS/Linux）；
   - 若输出为空：说明设备未监听该端口，重复步骤2重新设置。

#### 步骤4：针对安卓11+的特殊处理

安卓11及以上对无线调试做了限制，需开启「无线调试」专属开关：

1. 设备端进入「开发者选项」→ 找到「无线调试」（不是USB调试）→ 开启；
2. 点击「无线调试」→ 选择「使用配对码配对设备」；
3. 电脑端执行配对命令（按设备提示的配对码/端口）：
   ```bash
   adb pair 设备IP:配对端口 配对码
   # 示例：adb pair 192.168.1.100:44339 123456
   ```
4. 配对成功后，再执行：`adb connect 设备IP:5555`。

#### 步骤5：排除防火墙/杀毒软件干扰

1. Windows：关闭「Windows Defender防火墙」（临时测试），或允许adb.exe通过防火墙；
2. 第三方杀毒软件（如360、腾讯电脑管家）：暂时关闭「网络防护」「端口拦截」功能；
3. 设备端：关闭「网络隔离」「VPN」「代理」（这些会阻断局域网连接）。

### 三、快速验证是否解决

执行以下命令，若列表中显示「设备IP:端口 device」，则连接成功：

```bash
adb devices
```

### 总结

1. 「积极拒绝」的核心原因是**设备未监听指定端口**，优先重新插线执行 `adb tcpip 5555` 初始化；
2. 必须确保电脑和设备在**同一Wi-Fi**，且能ping通设备IP；
3. 安卓11+需额外开启「无线调试」并配对，而非仅依赖USB调试；
4. 端口占用/防火墙拦截是次要原因，可换端口或临时关闭防火墙测试。
